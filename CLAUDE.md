# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev              # Vite dev server (opens browser)
npm run build            # tsc + vite build → dist/
npm run extract-sprites  # Regenerate public/sprites/spritesheet.png from embedded PICO-8 gfx data
```

## Architecture

**Stack**: Vite + TypeScript + Kaplay (pixel-art game framework)

**Entry**: `src/main.ts` — initialises Kaplay, loads `loadSpriteAtlas`, registers scenes, starts `"game"`.

**Coordinate system**: The canvas is 512×512. One PICO-8 pixel = 4 world units (`DISPLAY_SCALE = 4` in `src/config.ts`). All sprites use `.scale(4)` so they render at 32×32 in the world. Physics constants in `config.ts` are pre-converted from PICO-8 per-frame values to world-units/s or world-units/s².

**Sprite sheet**: `public/sprites/spritesheet.png` is generated by `scripts/extract-sprites.ts` which decodes the `__gfx__` hex section from `wip.p8`. Character sprites are at gfx row 0 (indices 1–10); tile sprites at gfx row 7 (indices 112–113). The atlas names (`char1`–`char6`, `tile_block`, `tile_wall`) are defined in `src/main.ts`.

**Source layout**:
```
src/
  main.ts           kaplay init + atlas + scene registration
  config.ts         physics and display constants
  entities/
    player.ts       spawnPlayer() — body/area/sprite, movement, jump, char cycle
  scenes/
    game.ts         main game scene (gravity, player spawn, temp floor, camera)
    proof.ts        sprite loading smoke test (not the default scene)
scripts/
  extract-sprites.ts  one-shot PICO-8 → PNG converter
```
